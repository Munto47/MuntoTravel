# demo07 · 精细化交通规划

> **学习重点**：精细交通数据 vs. LLM 工具调用的架构选择；高德实时自驾 + 列车三分类数据库

## 与 demo06 的核心差异

| 维度 | demo06 | demo07 |
|------|--------|--------|
| **交通获取方式** | `plan_transport` @tool，LangGraph 自主调用 | `transport_client` 在 `main.py` 预先计算 |
| **自驾数据** | 单一方案，无高速路名 | 双策略（最快 + 少收费），提取途经高速路名 |
| **列车分类** | 混为"公共交通"一类 | 高铁(G/C) / 动车(D) / 普通列车(K/Z/T) 三类 |
| **班次详情** | 无 | 每类附带具体车次示例 + 发到时间 |
| **票价展示** | 粗略估算 | 按座位类型精确定价 |
| **前端交通卡片** | 混合卡片 | 自驾卡片（蓝色）+ 各类列车独立卡片 |
| **数据可靠性** | LLM 重新输出，可能 hallucination | 结构化数据直接返回前端，不经过 LLM |

## 架构设计要点

### 为什么交通改为"预算模式"？

```
demo06 （工具调用模式）
  main.py → run_graph() → LangGraph Agent → 调用 plan_transport tool → 返回文字 → LLM 引用

demo07 （预算注入模式）
  main.py → transport_client.get_transport_options()  ← 提前计算
          ↓                                          ↓
     transport_text (注入 LLM 上下文)          transport_detail (直接返回前端)
          ↓
       run_graph() → LangGraph Agent (只用 get_weather + get_attractions)
```

**选择依据**：
- 交通数据是"一次性确定的"（出发地+目的地固定），不需要 LLM 判断调用时机
- 列车数据来自本地数据库，结构化，前端需要精确渲染（价格表、班次表）
- 避免 LLM 重新输出结构化数据时可能的 hallucination

### 何时用工具调用，何时用预算注入？

| 使用工具调用 | 使用预算注入 |
|------------|------------|
| LLM 需要动态判断"是否需要查询" | 查询时机固定、可确定 |
| 数据用于叙事/推理 | 数据需要精确结构化展示 |
| 查询次数不确定（如多轮景点查询） | 查询一次即可完成 |
| 示例：`get_attractions`（按偏好查不同类别） | 示例：交通方案、汇率、固定参数 |

## 列车数据库覆盖范围

支持以下城市对（双向）：

| 城市对 | 列车类型 |
|--------|---------|
| 上海 ↔ 杭州 | 高铁(G/C) |
| 北京 ↔ 上海 | 高铁(G) + 普通列车夜铺(T/Z) |
| 北京 ↔ 西安 | 高铁(G) + 普通列车夜铺(K/Z) |
| 北京 ↔ 成都 | 高铁(G) + 长途卧铺(K) |
| 成都 ↔ 重庆 | 城际高铁(G/D) |
| 上海 ↔ 南京 | 高铁/动车(G/D) |
| 广州 ↔ 深圳 | 城际高铁(G/C) + 普通城际(T/K) |
| 北京 ↔ 天津 | 城际高铁(C/G) |
| 上海 ↔ 苏州 | 高铁(G/D) |
| 西安 ↔ 成都 | 西成高铁(G/D) |
| 广州 ↔ 桂林 | 动车/高铁(D/G) |
| 上海 ↔ 黄山 | 高铁/动车(G/D) |
| 北京 ↔ 青岛 | 高铁(G) |

## 自驾功能（高德 API）

有 `AMAP_API_KEY` 时：
1. Geocoding：城市名 → 经纬度
2. 策略0（速度优先/最快）：距离、时长、过路费、途经高速路名
3. 策略1（费用优先/少收费）：如果过路费差 > 15 元，显示第二方案

无 API Key 时：降级为 13 城市对的内置参考数据。

## 快速开始

```bash
cd demo07
cp .env.example .env    # 填入 API keys
pip install -r requirements.txt
python run.py           # http://localhost:8006
```

## API 端点

| 端点 | 说明 |
|------|------|
| `POST /api/questionnaire/analyze` | 分析问卷，返回用户画像（< 10ms） |
| `POST /api/trip/plan-with-profile` | 含画像的完整规划（含交通详情） |
| `POST /api/trip/plan` | 快速规划（无问卷，含交通详情） |
| `GET /api/transport?origin=X&destination=Y` | 单独查询交通方案（不调 LLM） |
| `GET /api/graph` | 返回 Mermaid 格式的 LangGraph 流程图 |

## 注意事项

- 列车时刻表为参考数据（每年12306会更新），实际出行请在12306确认
- 高德驾车 API 返回的是实时路况估算，节假日及早晚高峰会有明显差异
- 列车票价以官方标准价为准，春运/节假日无浮动（高铁固定价）
